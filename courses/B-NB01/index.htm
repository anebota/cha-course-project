Introduction to Computer Networks

What is a Network?

A network is a collection of interconnected devices that can communicate with each other. For example, a home network can connect devices like laptops, smartphones, and smart TVs to each other and the internet.

Everyday Uses of Networks:

A network can be used for accessing the internet, streaming media, online gaming, business communication, connecting smart home devices. For example, streaming a movie on Netflix requires your device to communicate with Netflix’s servers over the internet.

Types of Networks

Local Area Network (LAN)

This is a network that covers a small geographic area, like a home, office, or building. For example, the network in your home that connects your router to your computer and smartphone.

Wide Area Network (WAN)

This is a network that covers a large geographic area, such as a city, country, or the entire globe. For example, the internet, which connects networks across the world.

Personal Area Network (PAN)

A network for personal devices, typically within a range of a few meters. For example, Bluetooth connections between a smartphone and wireless headphones.

Network Hardware Components

These are the physical elements that can be used to set up a physical network among devices to enable them to communicate with each other.

  • Routers

    • These are devices that forward data packets between computer networks, directing traffic on the internet. For example, your home router connects your local network to your ISP’s network, enabling internet access.

  • Switches

    • These are devices that connect multiple devices within a LAN, using MAC addresses to forward data to the correct destination. For example, in an office, a switch connects multiple computers to form a single network.

  • Network Interface Cards (NICs

    • These are hardware components that enable a computer to connect to a network and provide the physical interface for networking. For example, the Ethernet port on your computer is part of the NIC.

  • Access Points

    • These are devices that allow wireless devices to connect to a wired network using Wi-Fi. For example, the wireless router in your home also acts as an access point or when you activate hotspot on your phone, it acts like an Access Point.

  • Ethernet Cables

    • Physical cables are used to connect devices within a LAN, providing a wired network connection. For example, the cable connecting your computer to your router or school switch. Cables used can be copper based or fiber optic cables depending on the application.

  • Wireless Network Cards

    • A wireless network card is essentially the hardware component that allows your device (like a computer, laptop, or smartphone) to connect to a wireless network. It's the bridge between your device and the world of Wi-Fi.

Networking Terms

Nodes

Nodes are devices connected to the network, such as printers, routers, or smartphones. i.e a printer that is connected to a network is a node.

Hosts

These are end devices that are assigned IP addresses and can send/receive data on a network. For example, your laptop is a host on your home network.

Clients and Servers

On a network, Clients request services or resources (e.g., a web browser on your computer), while Servers provide services or resources (e.g., a web server hosting a website). For example, when you access Google, your browser (client) requests the webpage from Google’s servers.

Network Addressing

MAC Addresses

A MAC (Media Access Control) Address is a unique identifier assigned to the network interface for communications within a network segment. This address is often written as 6 blocks of 12 hexadecimal numbers separated by a dash (-) or column (:). For example, 00:1A:2B:3C:4D:5E is a MAC address for a NIC. This address is unique for every device and no two devices in the world can have the same MAC address. The first 3 blocks of the MAC address identify the company issuing the device and the last 3 blocks identify the number of the device.

IP Addressing

Imagine you're sending a letter. You need an address to get it to the right place, right? Well, in the world of computers, that address is called an IP address. An IP address is a unique identifier assigned to every device connected to a network using Internet Protocol for communication. It's like a digital postal code for your computer, smartphone, or any other device that can connect to the internet. Think of it as the way computers find each other and communicate. There are two main types of IP addresses:
IPv4: An IPv4 address identifies a node on an IP network and consists of four numbers (each between 0 and 255)
IPv6: This is the newer version, designed to handle the growing number of devices connected to the internet. It uses a longer, more complex format.

  • Structure of an IPv4 Address

    • An IPv4 address identifies a node on an IP network and consists of four numbers (each between 0 and 255) separated by periods (e.g., 192.168.1.1). For example, 192.168.1.1 is a common default IP address for home routers. This address must be unique for each host connected to the same network.

  • Classes of IPv4 Addresses

    • IP addresses are divided into classes. The possible maximum number of devices that can be present on a particular network will determine the class of IPs the network should use.

    • Class A: For large networks (1.0.0.0 to 126.0.0.0)
    • Class B: For medium-sized networks (128.0.0.0 to 191.255.0.0)
    • Class C: For small networks (192.0.0.0 to 223.255.255.0)
    • Class D: For multicast groups (224.0.0.0 to 239.255.255.255)
    • Class E: Reserved for future use (240.0.0.0 to 255.255.255.255)

Types of IPv4 Addresses

  • Public IP Addresses: These IPs are routable on the internet, assigned by ISPs. You will also need to pay to use a public IP address
  • Private IP Addresses: These IPs are used within local networks, and are not routable on the internet.

    • Examples

    Class A Private:10.0.0.0 to 10.255.255.255

    Class B Private:172.16.0.0 to 172.31.255.255

    Class C Private:192.168.0.0 to 192.168.255.255 IPv6 Addressing
    • Structure of an IPv6 Address

      • An IP version 6 (IPv6) address consists of eight groups of four hexadecimal digits separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

    • Advantages of IPv6

      • IPv6 provides a larger address space enabling the connection of many more devices to the internet as a whole. It also provides enhanced security features and improved routing efficiency.

    Networking Protocols

    A networking protocol is a set of rules and standards that govern how devices communicate with each other on a network. It defines the format, order, and content of messages exchanged between network devices.

    • HTTP/HTTPS

      • HyperText Transfer Protocol and the secure version are protocols used for transmitting web pages over the internet. HTTP is unencrypted, HTTPS is encrypted for security. For example, accessing a secure website like https://www.bank.com uses HTTPS

    • DNS

      • All servers on the internet have IP addresses. Since humans are not good at recalling numbers, the Domain Name System is used to translate human-friendly domain names (e.g., www.example.com) into IP addresses. For example, typing “www.google.com” in your browser resolves to an IP address like 142.250.72.78.

    • DHCP

      • For devices to communicate with each other on an IP network, IP addresses must be configured on them. One way is to move from device to device to configure these addresses manually. Alternatively, a DHCP (Dynamic Host Configuration Protocol) server can be used to automatically assign IP addresses to devices that join the network. For example, when you connect to a Wi-Fi network, DHCP assigns your device an IP address.

    Common Networking Models

    A networking model is a conceptual framework that divides the complex process of network communication into smaller, more manageable layers or levels. It provides a standardized structure for understanding, designing, and troubleshooting networks.

    OSI Model

    The Open Systems Interconnection (OSI) model is the most well-known networking model. It divides network communication into seven layers, each with specific functions.

    Overview of the 7 Layers
    • Layer 1: Physical: Hardware transmission of raw data over a medium (e.g., cables).
    • Layer 2: Data Link: Handles error correction from the physical layer. Network switches operate in this layer
    • Layer 3: Network: Determines how data is sent to the receiving devices (IP addressing). Routers operate in this layer
    • Layer 4: Transport: Ensures complete data transfer (TCP/UDP).
    • Layer 5: Session: Manages sessions between applications.
    • Layer 6: Presentation: Translates data for the application layer.
    • Layer 7: Application: Provides network services to applications.

    When you send an email, each layer handles different aspects of the data transfer.

    Common Networking Models

    TCP/IP Model

    The TCP/IP model is a simplified version of the OSI model, specifically designed for the internet. It outlines how data is transmitted across networks. TCP/IP stands for Transmission Control Protocol/Internet Protocol, which are two core protocols used in this model.


    Unlike the OSI model, which has seven layers, the TCP/IP model consists of four layers.

    Overview of the 4 Layers
    • Layer 4: Application Layer: The top layer, responsible for user-facing applications and protocols like HTTP, FTP, and email. (equivalent to OSI Layers 5-7).
    • Layer 3: Transport Layer: Handles end-to-end communication, ensuring reliable data delivery. Protocols like TCP and UDP operate at this layer. (equivalent to OSI Layer 4).
    • Layer 2: Internet Layer: Routes packets across networks using IP addresses (equivalent to OSI Layer 3).
    • Network Access Layer: Handles physical transmission of data, including protocols for accessing the network medium

    The TCP/IP model is more practical and widely used in network engineering and administration.
